# チーム開発体験プロジェクト

GitHubを使ったチーム開発の実践体験用ウェブサイトです。

## プロジェクト概要

このプロジェクトは、[Craft Stadiumのチーム開発ハンズオンガイド](https://www.craftstadium.com/blog/team-development-handson-guide)に基づいて作成された体験用ウェブサイトです。

### 主な機能

- **ホームページ** (`index.html`) - プロジェクトの概要とメインページ
- **チーム紹介ページ** (`team.html`) - 開発チームメンバーの紹介
- **お問い合わせフォーム** (`contact.html`) - ユーザーからの問い合わせを受け付けるフォーム

## セットアップ手順

1. リポジトリをクローンする
```bash
git clone [リポジトリURL]
cd team-development
```

2. ローカルサーバーを起動する（任意）
```bash
# Python 3の場合
python -m http.server 8000

# Node.jsの場合
npx serve .
```

3. ブラウザで `http://localhost:8000` にアクセス

## チーム開発の流れ

### 1. 初期設定
- リポジトリの作成
- 作業分担の決定
- ブランチ戦略の策定

### 2. 開発作業
- 各機能をIssueとして登録
- 担当者を割り当て
- ブランチを作成して開発

### 3. レビューとマージ
- プルリクエストの作成
- コードレビューの実施
- マージの実行

## Pull Request テンプレート

このプロジェクトでは、効率的なレビュープロセスを促進するためにPull Requestテンプレートを用意しています。

### 利用可能なテンプレート

1. **基本テンプレート** (`.github/pull_request_template.md`)
   - 一般的な変更に使用
   - シンプルで使いやすい構成

2. **新機能追加テンプレート** (`.github/pull_request_template_feature.md`)
   - 新機能の追加時に使用
   - 要件や背景の詳細記載が可能

3. **バグ修正テンプレート** (`.github/pull_request_template_bugfix.md`)
   - バグ修正時に使用
   - 修正前後の比較が可能

### 使用方法

1. GitHubでPull Requestを作成する際、テンプレートが自動的に適用されます
2. 必要に応じてテンプレートの内容を編集してください
3. チェックリストの項目を確認してからレビューを依頼してください
